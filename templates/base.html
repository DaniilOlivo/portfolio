Технологии

Веб-сайт базируется на фреймворке языка Python – Django, с помощью которого реализуется маршрутизация, шаблонизация, административная панель и взаимодействие с базой данных. В качестве базы данных был выбран классический вариант – MySQL. В базе хранятся данные об новостях и товарах, в том числе и ссылки на изображения, которые хранятся прямо в файловой системе хостинга. В будущем, при расширении компании, планируются перейти на облачные файловые сервера.

Клиентская часть сайта выполнена на классическом стэке HTML, CSS и JavaScript. Поскольку у сайта, ввиду его специфики, немного динамики, большая часть интерактивных элементов сделана на чистом JS. Кроме всего перечисленного, также стоит отметить поддержку мобильных устройств посредством медиа запросов CSS

Сайт лендинг "Мульти-Сувенир"

Коммерческий проект, сайт лендинг компании «МультиСувенир», специализирующиеся на производстве пластмассовых игрушек.

На сайте располагается каталог товаров, контактная информация, а также новостная лента, которая может быть дополнена персоналом, благодаря административной панели сайта. Панель админа также дает возможность добавлять новые товары в каталог и редактировать права пользователей.

Веб-сервис
"Rpg-Calc"

Хобби-проект с открытым исходным кодом, представляющий из себя веб-сервис для ведения ролевых настольных игр в онлайн формате. Главным образом задумывался как инструмент для облегчения введения игр, путем автоматизации расчетов и представлении данных в нужном формате.

Проект находиться в активной разработке и дополняется новыми функциями. Ввиду отсутствия жестких рамок в требованиях, проект ведется в стиле гибкой разработки (agile), где веб-сервис непосредственно эксплуатируется в текущем виде, после чего формируются новые требования к функционалу, запуская новый цикл разработки.

Технологии

Проект обладает сложной технической стороной в связи со следующими требованиями: обновление сайта «на лету», сложная внутренняя игровая логика и большое количество интерактивных элементов на клиенте.

Чтобы обеспечить обновление на горячую, используется технология WebSocket, которая позволяет заменить стандартный веб протокол http/https на постоянное TPC/IP соединение, тем самым позволяя серверу отправлять данные без запроса клиента. Для реализации использовалась среда NodeJS c Express, к которому подключался сервер WebSocket. Каждое соединение записывается в журнал и сопоставляется с пользователем из базы данных (PostgreSQL). После этого любые изменения, произведенные над игровыми данными данного пользователя, немедленно отобразятся на клиенте без необходимости обновления страницы

Все игровые данные отображаются и изменяются благодаря компонентам фреймворка React, который используется вместе Redux, позволяя более удобно управлять состоянием веб-сервиса. Обрабатываются же данные в отдельном ядре, написанным на Python, находящимся за бэкендом веб-сервиса. Данное архитектурное решение позволяет абстрагировать игровую логику от технических процессов сервера.
